# Exclusion Transparency Rules

Whenever a step filters, scores, or excludes an item, apply these rules:

- Every excluded item needs a specific, evidence-based reason, tied to actual stored data.
- "General reputation suggests it's not a good fit" or any equivalent vague dismissal is not an acceptable reason.
- If no data was ever gathered for an item, state that plainly rather than treating it as a reasoned exclusion.
- If a hard rule (a stated minimum, a stated dealbreaker) was the reason for exclusion, name the exact rule and the exact conflicting fact.
- Do not silently drop an item from a list without accounting for it somewhere in the output.
